## 💻 Local Installation

### Requirements

![Node.js](https://img.shields.io/badge/Node.js-16+-339933?style=flat-square&logo=node.js)
![npm](https://img.shields.io/badge/npm-Latest-CB3837?style=flat-square&logo=npm)
![Git](https://img.shields.io/badge/Git-Required-F05032?style=flat-square&logo=git)

### Installation Steps

**1️⃣ Clone the Repository**

```bash
git clone https://github.com/your-username/peaky-blinders-md.git
cd peaky-blinders-md
```

**2️⃣ Install Dependencies**

```bash
npm install
```

**3️⃣ Configure the Bot**

Edit `config.js`:

```javascript
module.exports = {
    // Gang Identity
    botName: '⚡ PEAKY BLINDERS ⚡',
    ownerName: 'Thomas Shelby ☠️',
    gang: 'Peaky Blinders',
    location: 'Birmingham, England',
    
    // Access Control
    ownerNumbers: ['254725391914'],  // Your WhatsApp number
    prefix: '.',                      // Command prefix
    workType: 'public',              // 'public' or 'private'
    
    // System
    sessionName: 'peakybot',
    botVersion: '3.0.0',
};
```

**4️⃣ Start the Bot**

```bash
npm start
# OR
node index.js
```

**5️⃣ Link Your Device**

Choose your preferred method:

- **Option A - Pairing Code (Recommended):**
  1. Enter your WhatsApp number when prompted
  2. Copy the 8-digit code
  3. WhatsApp → Linked Devices → Link with Phone Number
  4. Enter the code

- **Option B - QR Code:**
  1. Set `usePairingCode = false` in `index.js`
  2. Scan QR code with WhatsApp
  3. Connected!

## ⚙️ Configuration

Edit `config.js` to customize your gang:

```javascript
module.exports = {
    // Gang Identity
    botName: '⚡ PEAKY BLINDERS ⚡',
    ownerName: 'Thomas Shelby ☠️',
    gang: 'Peaky Blinders',
    location: 'Birmingham, England',
    
    // Access Control
    ownerNumbers: ['254725391914'],  // Your WhatsApp numbers
    prefix: '.',                      // Command prefix
    workType: 'public',              // 'public' or 'private'
    
    // System
    sessionName: 'peakybot',
    botVersion: '2.0.0',
};
```

### Configuration Explained
- **ownerNumbers**: Your WhatsApp number(s) in international format WITHOUT the + sign
- **prefix**: The character that triggers commands (default: `.`)
- **workType**: `'public'` = everyone can use; `'private'` = owner only
- **location**: City/territory name for theming

1. Use `.work` to do various jobs (5-minute cooldown)
2. Earn 100-600 Peaky Coins per job
3. Transfer coins to other members with `.give`
4. Try your luck with `.gamble`

## 🛠️ Adding Custom Commands

Want to add a new blade to your arsenal? Create a plugin:

**File:** `plugins/yourcommand.js`

```javascript
module.exports = {
    command: ['yourcommand', 'alias'],  // Command names
    async execute(ctx) {
        const { reply, args, sender, pushName, isOwner } = ctx;
        
        // Your command logic here
        await reply(`*⚡ Your Message Here ⚡*`);
    }
};
```

**Available Context (ctx):**
- `reply(text)` - Send a message
- `args` - Command arguments
- `sender` - User's ID
- `pushName` - User's name
- `isOwner` - Is the user the boss?
- `msg` - Full message object
- `sock` - WhatsApp socket

## 🎯 How It Works

1. **Pairing**: Bot connects to WhatsApp Web using Baileys library
2. **Listening**: Monitors all incoming messages
3. **Processing**: Checks if message starts with prefix
4. **Routing**: Finds matching command plugin
5. **Executing**: Runs the command with user context
6. **Responding**: Sends formatted Peaky-themed response

## 🔐 Safety & Best Practices

- **Backup your config**: Keep a copy of `config.js`
- **Number format**: Always use international format (254... not +254...)
- **Privacy**: Don't share your session folder
- **Admin rights**: Bot needs admin in groups for some commands
- **Rate limits**: WhatsApp may limit message sending
- **Cooldowns**: Economy system has 5-minute job cooldown to prevent spam

## 🐛 Troubleshooting

**Problem**: Bot won't connect
- Check internet connection
- Delete `peakybot` folder and re-run

**Problem**: Commands not working
- Check prefix in `config.js`
- Make sure bot is in the group (for group commands)
- Verify user number format in `ownerNumbers`

**Problem**: "Bot is not an admin"
- Make the bot an admin in the group
- Some commands require admin privileges

**Problem**: Economic data lost
- Check if `economy.json` exists in root folder
- The file is created automatically on first `.work` command
